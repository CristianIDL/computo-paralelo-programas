## Creación de un semáforo

```
int Crea_semaforo(key_t llave,int valor_inicial)
{
   int semid=semget(llave,1,IPC_CREAT|PERMISOS);
   if(semid==-1)
   {
      return -1;
   }
   semctl(semid,0,SETVAL,valor_inicial);
   return semid;
}
```

Inicialmente tenemos una variable `semid`, a la cual le pediremos un semáforo con `semget`:
- el SO necesita la llave,
- cuántos semáforos quieres,
- después le pedimos cuales permisos

En caso de que no se pueda crear el semáforo nos salimos del proceso.

Una vez creado le asignamos el valor inicial en la llamada al sistema de `semctrl`:
- el identificador de semáforo
- qué posición del semáforo se va a asignar en suposición inicial (como solo llevamos un semáforo lo ponemos en su posición inicial, si tuvieramos  un millón debemos decirle en qué posición lo queremos _similar a un arreglo_)
- SETVAL es el tipo de acción que vamos a realizar

FInalmente retornamos `semid`.

Esto pide una llave y el estado que debe entrar el semáforo.

En `dato=RegionNoCritica();` tenemos

Este down como tal no existe en c. Nosotros tenemos que crear la función.

```
void down(int semid)
{
   struct sembuf op_p[]={0,-1,0};
   semop(semid,op_p,1);
}

```

Aquí el down recibe este semáforo. Lo primero que hay que hacer es configurar esta estructura, siendo una estructura ya definida en una vi

Se asignan con `op_p` por la palabra de down en irlandés, en honor a [[E.W Djixtra]], mientras que op_v es la palabra en irlandés para `up`.

En `struct sembuf op_p[]={0,-1,0};` lo que importa es el `-1` que significa disminuir en down. Si fuera un `+1` sería aumentar para el up.

Luego el `semop(semid,op_p,1);`:
- id del semáforo
- hacer down
- el 1 indica cuantas veces

Si no se pudo hacer down, el [[Sistema Operativo]] ya bloquea al proceso por sí mismo. En el caso de no haber hecho down se paasa a la [[Región Crítica]]

# Explicando Región Crítica


Es cuando el proceso entra a la memoria y graba sus cosa. Eso se hizo a través de memoria compartoda

```void RegionCritica(int dato)
{
   int id_memoria;
   int *dato_memoria;
   key_t llave_memoria;
   llave_memoria=ftok("Archivo_memoria",'m');
   id_memoria=shmget(llave_memoria,sizeof(int),IPC_CREAT|PERMISOS);
   dato_memoria=shmat(id_memoria,0,0);
   printf("\nEstoy en la region critica y mi pid es: %d\n",getpid());
   printf("\nEl dato guardado por el proceso anterior es: %d\n", *dato_memoria);
   *dato_memoria=dato;
   printf("\nNuevo dato guardado en la memoria compartida: %d\n",*dato_memoria);
   sleep(4);
}```

- id_memoria es donde se guarda el identificador de la memoria, no la memoria persé.
    - para el shared memory get `shmget` se requiere una llave (no la del semáforo, sino que una nueva)


- dato_memoria es la interfaz que nos sirve.

> La memoria RAM siempre se accede a través de direcciones de memoria, utilizando [[Apuntadores]] que nos permitirán acceder a la dirección del dato.
> Cuando tu no tienes apuntador, es muy local; mientras que c puede permitirte acceder a cualquier lado de la memoria

> Es por esto que C es muy inseguro, pero existen protecciones del [[Sistema Operativo]] que evitan que puedas acceder al [[Kernel]]

    - dato memoria accede a través de la llamada al sistema de `shmat` (shared memory attach) se pegue al dato memoria. 

A través de la llave podré meterme para modificar lo que yo quiera. En este momento que accedamos al dato_memoria ya todos los procesos podrán ver lo que está pasando. Si hubierea otro proceso viendo la memoria compartida lo reconocerá

> Es increíblemente rápido, hablamos de nanosegundos, es por esto que las arquitecturas de las [[GPU]] se trabajan con memoria compartida.

Una vez que ya entro aquí guardo el dato y salgo.

> La RAM no está pensada para que se lea un solo pedacito muy rápidamente. Por ende el Sistema Operativo tiene una restrucción que retiraría el programa. En el caso de nuestro programa nono es tan rápid, pero habría otros que no podrían aguantar.

Una recomendación es no leer el mismo pedazo de memoria compartida en poco tiempo. En este caso que es un programa didáctico se ve de esta forma, pero en las arquitecturas reales transcurre un tiempo entre la lectura de toda la memoria, por lo que no estamos con el mismo pedacito todo el rato.


## Repaso semáfotos

Cuando los semáforos vale 0 quiere decir que nadie puede avanzar. Es como los semáforos de las calles cuando están en rojo.

Se puede hacer down cuando el semáforo es mayor que 0. El sistema operativo se encarga de hacer esas comprobaciones, ustedes no tienen que hacer nada

Up es poner el semáforo en 1 otra vez.
